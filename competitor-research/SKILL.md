# 竞品全网调研

输入一个公司名称或网站，自动收集竞品信息并生成完整的竞品分析报告。

## 触发条件

当用户说以下内容时启动此技能：
- "调研这个竞品"
- "分析这家公司"
- "competitor research"
- "竞品分析"
- "帮我了解一下 [公司名]"

## 执行步骤

### 步骤 1：确定调研目标

从用户输入中识别：
- 公司名称
- 公司官网 URL
- 是否需要找相似竞品

### 步骤 2：公司概况调研

使用 WebSearch 和 WebFetch 收集以下信息：

**基础信息**
- 公司全称
- 成立时间
- 总部位置
- 员工规模
- 融资情况

**人员信息**
- 创始人/CEO
- 核心团队
- 最新招聘动态

**搜索策略**：
```
"[公司名] crunchbase"
"[公司名] linkedin company"
"[公司名] 融资 投资"
"[公司名] founder CEO"
```

### 步骤 3：产品定价调研

访问公司官网的定价页面，提取：

**产品功能**
- 核心功能列表
- 差异化特性
- 技术栈

**定价方案**
- 套餐名称和价格
- 免费版/试用期
- 企业定制选项
- 当前促销活动

**搜索策略**：
```
"[公司名] pricing"
"[公司名] plans features"
site:[公司域名] pricing
```

### 步骤 4：用户评价调研

收集用户反馈和市场口碑：

**评价来源**
- Product Hunt
- G2 / Capterra
- Trustpilot
- App Store / Play Store
- Twitter/X 讨论

**提取内容**
- 评分/星级
- 好评要点（Top 3）
- 差评要点（Top 3）
- 典型用户画像

**搜索策略**：
```
"[公司名] reviews" site:g2.com OR site:producthunt.com
"[公司名] 评价 优缺点"
```

## 数据存储

竞品报告保存在：`~/.claude/cache/competitor-reports/`

文件命名：`{公司名}-{日期}.md`

## 依赖工具

- **WebSearch**: 全网搜索公司信息
- **WebFetch**: 抓取定价页面、评价详情
- **Write**: 保存分析报告

## 限制说明

- 非上市公司的财务数据可能不完整
- 部分网站（如 LinkedIn）可能有访问限制
- 评价数据可能存在时效性问题
